최근 수많은 Agent 개발 툴들이 등장하고 있다. 그중에서도 Google이 공개한 **ADK (Agent Development Kit)** 는 가장 주목받는 오픈소스 중 하나로, Agent 생태계의 새로운 전환점을 제시하고 있다.

ADK는 이름 그대로 “Agent 개발을 위한 개발 키트”로, 단순한 SDK 수준을 넘어 **A2A (Agent to Agent)** 프로토콜을 기반으로 한 **표준화된 Agent 개발 프레임워크**를 제공한다. 이를 통해 개발자는 복잡한 Agent 간 통신 로직을 직접 구현하지 않아도 되며, 표준화된 방식으로 Agent를 설계하고 배포할 수 있다.


### ADK의 차별점

AI 에이전트 생태계는 현재 다양한 프레임워크들이 난립하고 있다. LangChain, AutoGen, Agno, Swarm, CrewAI 등 수많은 오픈소스가 존재하지만, 이들은 주로 단일 Agent의 기능 구현이나 워크플로우 중심의 조합에 초점을 맞춘다. 반면 **ADK는 Agent 간의 통신과 상호작용(A2A)** 을 표준화하는 데 초점을 두고 있다.

즉, **ADK는 단일 Agent 개발 도구가 아니라, 다수의 Agent들이 협력할 수 있는 인프라 레벨의 표준을 제공**한다는 점에서 차별화된다.

### ADK의 철학: 표준화된 협력

ADK의 철학은 명확하다 — **“AI Agent 간의 협력을 표준화한다.”**

기존의 Agent 시스템은 각자의 프레임워크 내에서만 작동하는 폐쇄적인 구조를 가지고 있었다. 하지만 ADK는 이를 **인터넷의 HTTP 프로토콜처럼, 모든 Agent들이 공통 언어로 대화할 수 있는 환경**으로 확장한다. 이 덕분에, 서로 다른 팀이나 조직이 만든 Agent들도 동일한 A2A 프로토콜을 통해 쉽게 연동할 수 있다.
